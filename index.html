设置终端代理

终端设置代理

windows

  1. http
    set http_proxy=http://127.0.0.1:8118
    set https_proxy=http://127.0.0.1:8118
  2. socks5
    set http_proxy=socks5://127.0.0.1:1080
    set https_proxy=socks5://127.0.0.1:1080
  3. set http_proxy=

Linux 代理

  1. http
    export http_proxy=http://127.0.0.1:8118
    export https_proxy=http://127.0.0.1:8118
  2. socks5
    export http_proxy=socks5://127.0.0.1:1080
    export https_proxy=socks5://127.0.0.1:1080
  3. 终端中的 wget、curl 等都走 SOCKS5 代理
    export ALL_PROXY=socks5://127.0.0.1:1080
  4. unset http_proxy

Git 代理

  1. HTTP
    git config –global http.proxy http://127.0.0.1:8118
    git config –global https.proxy http://127.0.0.1:8118
  2. SOCKS5
    git config –global http.proxy socks5://127.0.0.1:1080
    git config –global https.proxy socks5://127.0.0.1:1080
  3. 取消
    git config –global –unset http.proxy
    git config –global –unset https.proxy

pip 设置代理

pip install -r requirements.txt –proxy=代理服务器IP:端口号

curl

curl -o home.html http://www.sina.com.cn

curl -O http://www.mydomain.com/linux/index.html

curl -x 10.10.90.83:80 -o home.html http://www.sina.com.cn

curl -e http://localhost http://www.sina.com.cn 伪造来源地址

一种以 .service 结尾的单元（unit）配置文件，

linux 编译安装opencv

Linux 编译OpenCV

安装

  1. 安装类库
    sudo apt-get install cmake git libgtk2.0-dev pkg-config libavcodec-dev libavformat-dev libswscale-dev

sudo apt-get install python-dev python-numpy libtbb2 libtbb-dev libjpeg-dev libpng-dev libtiff-dev libjasper-dev libdc1394-22-dev

  1. cmake

cmake -DCMAKE_BUILD_TYPE=RELEASE -DCMAKE_INSTALL_PREFIX=/usr/local -DOPENCV_GENERATE_PKGCONFIG=ON ..

最后一个参数会生成 /usr/local/lib/pkgconfig

  1. 编译
    make -j8

  2. 安装
    make install

pkgconfig 运行

默认情况,pkg-config首先在 /usr/lib/pkgconfig/ /usr/local/lib/pkgconfig 查找xxx.pc
没有的话在 PKG_CONFIG_PATH

pkg-config –libs –cflags opencv
查看gcc的CFLAGS参数

Linux 变量

C_INCLUDE_PATH C 头文件
CPLUS_INCLUDE_PATH C++ 头文件
LD_LIBRARY_PATH 动态链接库路径
LIBRARY_PATH 静态库的路径

运行opencv

  1. 直接指定 -I —L -l
    g++ test.cpp -I /usr/local/include/opencv4 -lopencv_features2d -lopencv_calib3d -opencv_imgproc -lopencv_core -lopencv_highgui -lopencv_imgcodecs -o test

缺少lib可以用下面的方法
nm -A lib64/libopencv*.a |grep “头文件”

  1. pkg-config
    pkg-config –help
    pkg-config –modversion opencv4
    g++ test.cpp pkg-config --libs --cflags opencv4
    生成a.out

  2. cmake CMakeList.txt
    cmake_minimum_required(VERSION 2.8)
    project(opencv)
    set(OpneCV_DIR /path/to/opencv-x86/)
    find_package(OpenCV REQUIRED)
    add_executable(opencv try.cpp)
    target_link_libraries(opencv ${OpenCV_LIBS})

  3. 动态链接库
    /etc/ld.so.conf
    include /etc/ld.so.conf.d/*.conf

sudo ldconfig

Debain版本

Debian 一个Linux发行版

Debian的前世今生

Debian名字的由来—-DebianGNU/Linux是由一个叫做伊恩·默多克（IanMurdock）在1993年发起的，他的名字以Ian开头，他太太的名字Debra开头三个字母是Deb。
Debian是社区类Linux的典范，是迄今为止最遵循GNU规范的Linux系统。
Debian最早由 Ian Murdock于1993年创建，分为三个版本分支： stable(服务器版), testing(稳定版) 和unstable(测试版)。

Debian的命名

Debian 起源于1993年8月16日，由Ian Murdock创建（Debian History）。
每一代的版本代号（codename）是以玩具总动员（Toy Story）的成员命名的。
详细见Debian官网
